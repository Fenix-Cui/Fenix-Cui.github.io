设计方法原则

保持方法的原子性：一个方法只实现一个功能(有利于后期扩展)

return的作用不只是返回值，还可以终止方法

值传递：传递的是值，改变形参不会影响实参，因为二者在内存中地址不同

引用传递：传递的是地址，改变形参会影响实参，因为二者指向同一地址

注：Java都是值传递

JVM是根据方法的参数类型来判断用的哪一个方法
